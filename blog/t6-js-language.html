JS Language

What are all the use cases for:

Parentheses ()

One of the uses for parentheses in Javascript is to group something, just like with normal mathemetics. Whatever is between the parentheses takes prevelance. Parentheses can also be used to contain an expression or to contain a list of parameters.

Example:

int a;
a = (4 +3) * 2;          // group: the sum of 4 and 3 is calculated first
if a (a > 10) {           // contain an expression
line (a, 0, a, 100);    // contain a list of parameters
}

Another important use for parentheses is with 'functions'. A function is a contained piece of code that performs a particular task. A function always has parentheses behind it. They either hold parameters, to be used within the function, or they can be empty, which means there is no parameter input for the function. It looks like this:

function (parameter0, parameter1)
or
function ()

Brackets [ ]

Brackets are used with arrays. An array is an object that can be used to store multiple values in one single variables. The array object, will display those values between brackets. When you want to access a value in the array, you use the brackets as well, with a number that identifies that value within that array.

Example:

var pets = ["cats", "dogs", "guinea pigs"]    //array with three values, numbered 0, 1 and 2

Accessing this array also uses brackets:

pets[2] = "guinea pigs"

Braces { }

Braces are used to group lines of code for 'control flow'. Within the braces you can place statements, like for-loops, if-else's or define functions. They are also used to define object literals, which are comma separated lists of name/value pairs, wrapped within the braces.

Single quotes ' ' and Double quotes " "

Single and double quotes can both be used to define strings of text. It's best to be consistent though, only use one or the other. If you use single quotes and your string contains a word with a single quote, ie 'Richard's code' you need to include a backslash: 'Richard\'s code'. The backslash prevents javascript from picking up the second ' as the end of the string, but instead interprets it being part of the string.
